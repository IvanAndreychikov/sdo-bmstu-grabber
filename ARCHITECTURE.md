# Архитектура проекта sdo-bmstu-grabber

Назначение: авторизоваться в СДО МГТУ (Moodle), обойти секции курса начиная
с заданной, и сохранить на диск **видео (rutube, максимальное качество)** и
**презентации**, сохранив названия и структуру курса в виде папок.

## Стек и зависимости

- **Python 3.13**, изолированный `venv/` в корне проекта.
- `requests` + `beautifulsoup4` + `lxml` — авторизация и парсинг Moodle.
- `yt-dlp` — скачивание rutube (понимает приватные embed-URL, сам выбирает
  лучшее качество, мерджит HLS).
- `imageio-ffmpeg` — поставляет статический бинарник `ffmpeg` прямо в venv
  (без прав администратора), нужен yt-dlp для ремукса в mp4.

## Принятые решения

- **Разделение ответственности.** Сетевой слой (`MoodleClient`), парсинг
  (`CourseParser`), скачивание видео и файлов — отдельные классы. Оркестратор
  не знает деталей HTML или yt-dlp, только дёргает интерфейсы.
- **Креды не хардкодим в коде.** Источники по возрастанию приоритета:
  defaults → `config.json` → переменные окружения → аргументы CLI.
  `config.json` добавлен в `.gitignore`.
- **Структура на диске** = структура курса:
  `result/<NN> <Имя секции>/` — папка на каждую секцию (префикс `NN` =
  номер секции Moodle с ведущим нулём, чтобы сортировка на диске совпадала с
  порядком курса). Внутри — видео `NN - <Название>.mp4` и файлы презентаций
  с оригинальными именами. Нумерация элементов = порядок на странице секции.
- **Максимальное качество видео** — селектор yt-dlp `best` (rutube отдаёт
  единые muxed-потоки, лучший = максимальное разрешение, до 720p в примерах).
- **Идемпотентность.** Уже скачанные файлы пропускаются (`skip_existing`),
  чтобы длинный прогон можно было перезапускать без перекачивания.
- **Имена файлов** чистятся под Windows (`sanitize_filename`): убираются
  `<>:"/\|?*`, зарезервированные имена, хвостовые точки/пробелы, длина.
- **Названия видео** берём из метаданных rutube (через yt-dlp) — в самом
  Moodle-label осмысленных подписей к iframe нет.
- **ffprobe.** `imageio-ffmpeg` поставляет только `ffmpeg`, без `ffprobe`.
  Это не мешает: rutube-варианты уже muxed, мы берём один лучший поток, и
  yt-dlp скачивает/ремуксит в mp4 без ffprobe. Предупреждение про ffprobe
  глушится в логе (`_YdlLogger`).

## Статус проверки

Сквозной прогон на секции 3 пройден успешно:
- логин в Moodle — ок;
- карта курса — 14 учебных секций (3..16);
- разбор секции 3 — 5 видео + 1 презентация, порядок сохранён;
- презентация скачана (PDF, ~1.5 МБ, имя `06 - ...LSTM.pdf`);
- видео скачано и проверено `ffmpeg -i`: **1280x720**, h264+aac, 31:31,
  валидный mp4 (~104 МБ) — берётся максимальное качество.
