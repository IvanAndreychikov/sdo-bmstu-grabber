# sdo-bmstu-grabber

*Доступно на других языках: [English](README.md).*

Скачивает видео-лекции (rutube) и презентации курса из СДО МГТУ им. Баумана
(`sdo.bmstu.ru`, Moodle), сохраняя названия и структуру курса в виде папок.

- Видео берутся в **максимальном доступном качестве**.
- Структура на диске повторяет структуру курса: одна папка на секцию.
- Повторный запуск **не перекачивает** уже скачанные файлы.

Подробности по устройству сайта — в [SITE_STRUCTURE.md](SITE_STRUCTURE.md),
по архитектуре кода — в [ARCHITECTURE.md](ARCHITECTURE.md).

## Быстрый старт

Все шаги от клонирования до запуска (команды для **cmd** на Windows):

```cmd
:: 1. Склонировать репозиторий и перейти в папку проекта
git clone <URL-репозитория> sdo-bmstu-grabber
cd sdo-bmstu-grabber

:: 2. Скопировать шаблон конфига
copy config.example.json config.json

:: 3. Открыть config.json и вписать свои логин/пароль от СДО
::    (при желании поменять output_dir, start_section и т.д.)
notepad config.json

:: 4. Создать виртуальное окружение
py -m venv venv

:: 5. Установить зависимости (ffmpeg ставить отдельно НЕ нужно —
::    он приходит пакетом imageio-ffmpeg прямо в venv)
venv\Scripts\python.exe -m pip install -r requirements.txt

:: 6. Запустить — скачает весь курс с темы section=3 до конца в папку result\
venv\Scripts\python.exe main.py
```

Вместо правки `config.json` креды и параметры можно задать переменными
окружения (`SDO_USERNAME`, `SDO_PASSWORD`, `SDO_OUTPUT_DIR`, ...) или
аргументами командной строки (см. ниже).

## Дополнительные варианты запуска

```cmd
:: одна секция (для проверки)
venv\Scripts\python.exe main.py --only-section 3

:: своя папка назначения и без пропуска уже скачанного
venv\Scripts\python.exe main.py --output-dir "D:\courses\dl" --no-skip

:: задать креды прямо в команде, без config.json
venv\Scripts\python.exe main.py --username ivan.andreychikov --password ***
```

### Полезные флаги

| Флаг | Назначение |
|---|---|
| `--start-section N` | с какой секции начинать (по умолчанию 3) |
| `--end-section N` | на какой секции закончить (по умолчанию — последняя) |
| `--only-section N` | обработать ровно одну секцию |
| `--output-dir PATH` | куда складывать файлы |
| `--no-skip` | перекачивать даже уже существующие файлы |
| `-v` | подробный лог |

## Результат

```
result/
└── 03 1. Рекуррентные нейронные сети. LSTM слои/
    ├── 01 - <название видео>.mp4
    ├── 02 - <название видео>.mp4
    ├── ...
    └── 06 - Рекуррентные нейронные сети. LSTM.pdf
```

По умолчанию всё складывается в папку `result/` в корне проекта (она в
`.gitignore`). Видео (`.mp4`) и презентации (`.pdf`/`.pptx`) лежат рядом в
папке секции — это отдельные файлы.
