# sdo-bmstu-grabber

*Доступно на других языках: [English](README.md).*

Скачивает видео-лекции, вебинары, презентации, приложенные файлы (например,
ноутбуки с кодом) и текстовые/тестовые страницы из СДО МГТУ им. Баумана
(`sdo.bmstu.ru`, Moodle), сохраняя названия видео и полную структуру курса в
виде папок.

- Видео берутся в **максимальном доступном качестве** — и rutube-вставки (как на
  странице секции, так и внутри уроков `mod/page`), и видео с сервера Moodle.
- **Вебинары** (записи MTS Link / webinar.ru) скачиваются и склеиваются в одно
  видео «картинка-в-картинке»: демонстрация экрана (или загруженная презентация
  спикера) как основное изображение, а камера спикера — в правом верхнем углу
  (при наличии NVIDIA используется NVENC, иначе — CPU).
- **Глубоко вложенные секции** обрабатываются: дерево курса обходится рекурсивно,
  так что модули → темы → отдельные уроки находятся и повторяются вложенными
  папками.
- **Текстовые страницы, тесты и обратная связь** сохраняются в **PDF** (без
  запуска попыток прохождения тестов), чтобы не потерять текстовый материал.
- Приложенные ссылки (файлы на Google Drive и т.п.) скачиваются рядом с видео;
  если файл скачать не удалось — сохраняется `.url`-ярлык.
- Повторный запуск **не перекачивает** уже скачанные файлы.

> Курс выбирается параметром `course_id` (и `start_section`) в `config.json` или
> флагами `--course-id` / `--start-section`. Например, `course_id=86` — курс с
> предзаписанными уроками; `course_id=297` — курс с вебинарами «Data Science PRO».

Подробности по устройству сайта — в [SITE_STRUCTURE.md](SITE_STRUCTURE.md),
по архитектуре кода — в [ARCHITECTURE.md](ARCHITECTURE.md).

## Быстрый старт

Инструмент кросс-платформенный (Windows, macOS, Linux). Все шаги от
клонирования до запуска:

**Windows (cmd):**

```cmd
:: 1. Склонировать репозиторий и перейти в папку проекта
git clone https://github.com/IvanAndreychikov/sdo-bmstu-grabber sdo-bmstu-grabber
cd sdo-bmstu-grabber

:: 2. Скопировать шаблон конфига
copy config.example.json config.json

:: 3. Открыть config.json и вписать свои логин/пароль от СДО
::    (при желании поменять output_dir, start_section и т.д.)
notepad config.json

:: 4. Создать виртуальное окружение
py -m venv venv

:: 5. Установить зависимости (ffmpeg ставить отдельно НЕ нужно —
::    он приходит пакетом imageio-ffmpeg прямо в venv)
venv\Scripts\python.exe -m pip install -r requirements.txt

:: 6. Запустить — скачает весь курс с темы section=3 до конца в папку result\
venv\Scripts\python.exe main.py
```

**macOS / Linux (bash):**

```bash
# 1. Склонировать репозиторий и перейти в папку проекта
git clone https://github.com/IvanAndreychikov/sdo-bmstu-grabber sdo-bmstu-grabber
cd sdo-bmstu-grabber

# 2. Скопировать шаблон конфига
cp config.example.json config.json

# 3. Открыть config.json и вписать свои логин/пароль от СДО
#    (при желании поменять output_dir, start_section и т.д.)
nano config.json            # или vim / любой редактор

# 4. Создать виртуальное окружение
python3 -m venv venv

# 5. Установить зависимости (ffmpeg приходит пакетом imageio-ffmpeg в venv)
venv/bin/python -m pip install -r requirements.txt

# 6. Запустить — скачает весь курс с темы section=3 до конца в папку result/
venv/bin/python main.py
```

> На Linux убедитесь, что в окружении используется UTF-8 локаль
> (например, `LANG=C.UTF-8` или `…UTF-8`) — иначе кириллические имена папок
> курса/секций могут записаться некорректно. На macOS и большинстве
> десктопных дистрибутивов Linux UTF-8 по умолчанию.

Вместо правки `config.json` креды и параметры можно задать переменными
окружения (`SDO_USERNAME`, `SDO_PASSWORD`, `SDO_OUTPUT_DIR`, ...) или
аргументами командной строки (см. ниже). На macOS/Linux используйте
`venv/bin/python` там, где в примерах ниже указан `venv\Scripts\python.exe`.

## Дополнительные варианты запуска

```cmd
:: одна секция (для проверки)
venv\Scripts\python.exe main.py --only-section 3

:: своя папка назначения и без пропуска уже скачанного
venv\Scripts\python.exe main.py --output-dir "D:\courses\dl" --no-skip

:: задать креды прямо в команде, без config.json
venv\Scripts\python.exe main.py --username my-personal-login --password my-personal-password
```

### Полезные флаги

| Флаг | Назначение |
|---|---|
| `--course-id N` | какой курс качать (по умолчанию из конфига) |
| `--start-section N` | с какой секции начинать (по умолчанию 3) |
| `--end-section N` | на какой секции закончить (по умолчанию — последняя) |
| `--only-section N` | обработать ровно одну секцию |
| `--output-dir PATH` | куда складывать файлы |
| `--concurrency N` | сколько параллельных соединений на один файл (по умолчанию 4) |
| `--max-name-length N` | макс. длина имени папки/файла в символах (по умолчанию 36, чтобы путь укладывался в лимит Windows MAX_PATH); `0` — без обрезки |
| `--no-skip` | перекачивать даже уже существующие файлы |
| `-v` | подробный лог |

```
│   │   ├── 01 - <название видео>.mp4
│   │   ├── ...
│   │   └── 06 - Рекуррентные нейронные сети. LSTM.pdf
│   └── 06 3. Обзор библиотеки PyTorch.../
│       ├── 01 - <название видео>.mp4
│       └── 06 - 11.8 PyTorch.ipynb        # приложенный файл с Google Drive
└── Наука о данных профессиональный уровень (Data Science PRO)/   # course_id=297
    ├── 07 Вебинары 16748/
    │   ├── 01 - Вебинар 1 от 27.01.2026.mp4     # склеенный вебинар (PiP)
    │   └── ...
    └── 12 Материалы курса/                       # глубоко вложенные секции
        └── 01 Модуль 1/
            └── 01 Тема 1. Введение в Big data…/
                └── 01 1.1. Введение в предмет/
                    ├── 01 - Data Science, 1 часть.mp4    # видео из mod/page
                    ├── 02 - 1.1 Введение в Big Data.pdf  # презентация
                    └── 01 - Тест 1.pdf                   # тест сохранён в PDF
```

По умолчанию всё складывается в папку `result/` в корне проекта (она в
`.gitignore`), внутри — одна корневая папка с названием курса. Видео (`.mp4`),
вебинары (`.mp4`), презентации (`.pdf`/`.pptx`) и приложенные файлы (`.ipynb` и
т.п.) лежат рядом в папке секции — это отдельные файлы.
